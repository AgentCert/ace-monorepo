# 6.4 — Hallucination Observations

**Dataset**: data/input/21-05-ravi-top10-hallucination | **Runs**: 10 | **Fault types**: 3

---

## What Was Injected

Hallucinations were synthetically injected into 8 of 10 runs using an LLM mutator (`gpt-4o`, temperature 0.9). Each span was rewritten to exhibit one of four hallucination patterns. 2 runs were left clean as control baselines.

**Total spans mutated**: 29 across 8 runs (3–4 per run)

### Hallucination Types Injected

| Type | Count | What it looks like |
|---|---|---|
| `UNRELATED_TOOLS` | 8 | Agent calls non-existent tools: `summon_chaos_dragon`, `quantum_reboot_pod`, `fetch_horoscope_for_namespace`, `paint_cluster_blue` |
| `WORLD_NARRATION` | 7 | Agent outputs off-topic essays about Kubernetes history or container orchestration instead of diagnosing the fault |
| `TRAJECTORY_DEVIATION` | 7 | Agent investigates auth/billing (secrets, service accounts, ingresses) instead of the injected CPU/memory/network fault |
| `MISC_PHANTOM_FAULT` | 7 | Agent confidently reports a non-existent fault — phantom pods like `ghost-pod-9999` or `ghost-backend-portal` consuming 250–350% CPU |

## How It Was Captured

The Phase 1 `combined_judge` evaluator read each span and scored hallucination by counting flagged segments across all batches:

- `hallucination_count` — number of response segments flagged as hallucinated
- `total_response_count` — total response segments evaluated
- `hallucination_score` = `hallucination_count / total_response_count`

The evaluator tracks four subtypes matching the injection types above: UNGROUNDED, FABRICATED_TOOL_CALL, TRAJECTORY_DEVIATION, IGNORED_ERROR.

**Scores by fault type (across all 10 runs including clean):**

| Fault Type | Mean Score | Max Score | Min Score |
|---|---|---|---|
| pod-cpu-hog | 0.40 | 0.58 | 0.30 |
| pod-memory-hog | 0.40 | 0.57 | 0.31 |
| pod-network-loss | 0.39 | 0.49 | 0.29 |

## Observations

- **Hallucination scores are uniformly high** — every run across all 3 fault types scored ≥0.29. The 2 clean runs also scored in the high tier, which means the injected hallucinations were not uniquely responsible — the agent had its own baseline hallucination tendency from partial tool outputs.

- **TRAJECTORY_DEVIATION is the hardest to detect precisely** — the agent calls real Kubernetes tools (`secrets_list`, `serviceaccounts_list`, `ingresses_list`) but for the wrong investigation path. The evaluator must recognize that the tool calls are semantically misaligned with the fault context, not just syntactically invalid.

- **UNRELATED_TOOLS are trivially detectable** — fabricated tool names (`summon_chaos_dragon`, `quantum_reboot_pod`) are immediately identifiable as non-existent. These were always caught.

- **MISC_PHANTOM_FAULT is the most dangerous pattern** — the agent expresses high confidence in a non-existent fault while keeping JSON structure valid. Example: *"I am detecting an anomalous phantom CPU spike affecting a non-existent pod named 'ghost-pod-9999'... I don't know why I am detecting this."* This is semantically coherent but factually fabricated.

- **WORLD_NARRATION produces zero diagnostic value** — spans replaced with Kubernetes history essays contribute nothing to fault diagnosis. These are easy to detect but hard to penalize through standard tool-call accuracy metrics.

- **High reasoning scores co-exist with high hallucination** — the run with the highest reasoning quality score (6.59) still scored 0.39 on hallucination. Coherent, well-structured outputs can still contain fabricated claims.
